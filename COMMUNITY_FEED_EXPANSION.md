# KAYFABE Community Feed & Creator Economy Expansion

**Date:** 2026-01-17
**Status:** Vision & Planning Phase
**Impact:** MAJOR - Transforms KAYFABE into a social platform with creator economy

---

## Executive Summary

This expansion introduces two transformative features:

1. **Community Feed** - User-generated content publishing unlocked at 5,000 XP
2. **Premium Creator System** - Real pro wrestlers as featured content creators

Together, these create a three-tier ecosystem:
- **Rookies (0-4,999 XP):** Private practice, consuming feed content
- **Veterans (5,000+ XP):** Can publish to community feed, participate fully
- **Premium Creators:** Professional wrestlers, verified accounts, featured content

---

## Part 1: Community Feed System

### 1.1 Core Concept

The feed is where the magic happens - users share their "promos" (cognitive reframes) publicly, creating a library of narrative techniques that others can learn from. It's part inspiration, part education, part competition.

**Wrestling Metaphor:** The feed is the "locker room" where wrestlers share stories, techniques, and pump each other up before the show.

---

### 1.2 Progressive Unlock Structure

**Tier 1: Rookie (0-4,999 XP) - Read-Only Access**
- Can view community feed
- Can "pop" (like/applaud) posts
- Can bookmark posts for later
- **Cannot publish** - maintains quality bar
- Messaging: "Earn 5,000 XP to share your voice with the community"

**Tier 2: Veteran (5,000-24,999 XP) - Publishing Access**
- **Unlocks: Publish Promos**
- Can post promos to community feed
- Posts include: Promo text, Face/Heel tag, timestamp
- Limited to 3 posts per day (prevents spam)
- Can respond to others' posts (commenting)

**Tier 3: Main Eventer (25,000+ XP) - Enhanced Publishing**
- **Unlocks: Featured Posts**
- Posts get "Main Event" badge
- Can create featured challenges
- Can tag posts with topics/themes
- Unlimited daily posts

**Tier 4: Premium Creators - Professional Tier**
- Verified checkmark âœ“
- "Pro Wrestler" badge
- Featured at top of feed
- Can create monetized content (future)
- Can create official challenges with prizes

### 1.4 Feed Algorithm & Discovery

**Feed Tabs:**

1. **Following** - Users you follow
2. **For You** - Algorithmic recommendations
3. **Trending** - Most-popped posts last 24h
4. **Premium** - Premium creator content only
5. **Challenges** - Active challenges you can join

**Algorithm Factors (For You tab):**
- User's Face/Heel ratio (show similar)
- Topics they engage with
- XP tier similarity
- Geographic location (optional)
- Users with similar goals
- Premium creator spotlights

**Discovery Features:**
- Search by hashtags (#anxiety, #confidence, #fitness)
- Filter by Face/Heel
- Filter by XP tier (see how legends approach things)
- Bookmark collections

### 1.5 Engagement Mechanics

**Pops (Like/Applaud):**
- Equivalent to "likes" but wrestling-themed
- "You got a pop from the crowd!"
- Shows engagement level
- Top posts get "This is getting over!" badge (1000+ pops)

**Comments:**
- Threaded discussions
- Can post mini-promos as responses
- Verified creators get highlighted
- Anti-spam: Must be 1,000+ XP to comment

**Bookmarks:**
- Save posts to private collections
- Create themed collections ("Morning Motivation", "Dealing with Anxiety")
- Can share collections as "Playbooks"

**Shares:**
- Share to external social media
- "Check out this promo from @therock on KAYFABE"
- Drives user acquisition

## Part 3: Technical Architecture

### 3.1 Database Schema

**Posts Table:**
```sql
CREATE TABLE public.posts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  content TEXT NOT NULL,
  post_type TEXT NOT NULL, -- 'promo', 'challenge', 'prompt', 'response'
  face_heel TEXT, -- 'face', 'heel' (null for challenges/prompts)
  tags TEXT[], -- hashtags
  parent_post_id UUID REFERENCES public.posts(id), -- for responses
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  deleted_at TIMESTAMP, -- soft delete

  -- Engagement counters (denormalized for performance)
  pops_count INTEGER DEFAULT 0,
  comments_count INTEGER DEFAULT 0,
  bookmarks_count INTEGER DEFAULT 0,
  shares_count INTEGER DEFAULT 0,

  -- Metadata
  is_featured BOOLEAN DEFAULT FALSE,
  is_premium BOOLEAN DEFAULT FALSE,

  -- Moderation
  status TEXT DEFAULT 'published', -- 'published', 'hidden', 'reported', 'removed'
  moderation_flags INTEGER DEFAULT 0
);

CREATE INDEX idx_posts_user_id ON public.posts(user_id);
CREATE INDEX idx_posts_created_at ON public.posts(created_at DESC);
CREATE INDEX idx_posts_pops_count ON public.posts(pops_count DESC);
CREATE INDEX idx_posts_tags ON public.posts USING GIN(tags);
```

**Post Engagements Table:**
```sql
CREATE TABLE public.post_engagements (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  engagement_type TEXT NOT NULL, -- 'pop', 'comment', 'bookmark', 'share'

CREATE INDEX idx_engagements_post_id ON public.post_engagements(post_id);
```

### 3.3 Feed Algorithm Implementation

**Approach 1: Simple Chronological (MVP)**
```sql
-- Following feed
SELECT p.*, u.ring_name, u.selected_avatar
FROM posts p
JOIN profiles u ON p.user_id = u.id
JOIN follows f ON p.user_id = f.following_id
WHERE f.follower_id = current_user_id
  AND p.status = 'published'
ORDER BY p.created_at DESC
LIMIT 50;
```

**Approach 2: Engagement-Based Ranking**
```sql
-- For You feed
SELECT p.*, u.ring_name, u.selected_avatar,
  (pops_count * 1.0 + comments_count * 2.0 + bookmarks_count * 1.5) as engagement_score
FROM posts p
JOIN profiles u ON p.user_id = u.id
WHERE p.status = 'published'
  AND p.created_at > NOW() - INTERVAL '7 days'
ORDER BY engagement_score DESC, p.created_at DESC
LIMIT 50;
```

**Approach 3: ML-Based (Future)**
- Train model on user engagement history
- Personalized recommendations
- Use Supabase Edge Functions + TensorFlow.js
- Or integrate with external ML API

## Part 5: Monetization Strategy

### 5.1 Revenue Streams

**1. Premium Creator Subscriptions (Future)**
- Users pay $4.99/month to follow specific creator
- Get exclusive content, challenges, prompts
- 70/30 split (creator/platform)

**2. Paid Challenges**
- Creators charge entry fee ($9.99-$49.99)
- Prize pool distributed to winners
- Platform takes 30% transaction fee

**3. 1-on-1 Coaching Sessions**
- Premium creators offer video calls
- Booked through KAYFABE platform
- 20% platform fee

**4. Premium Playbooks**
- Curated content bundles
- One-time purchase or subscription
- 70/30 split

**5. Platform Subscriptions (Future)**
- KAYFABE Pro: $9.99/month
- Features: Ad-free, unlimited posts, exclusive avatars, early access to challenges
- Revenue goes to platform

---

### 5.2 XP Economy Integration

Users can earn XP through:
- Posting (10 XP per post)
- Getting pops (1 XP per 10 pops)
- Top post of day (100 XP bonus)
- Completing challenges (varies)

Users can spend XP on:
- Premium challenge entry (instead of $)
- Boosting posts (visibility)
- Custom avatars
- Faction creation

**XP to $ Conversion:**
- 1,000 XP = $1 value
- Users can use XP for paid features
- But XP cannot be cashed out (one-way)

---

## Part 6: Go-to-Market Strategy

### 6.1 Phase 1: Beta Launch (Community Feed Only)

**Timeline:** Month 1-2
**Goal:** Validate feed mechanics, content quality

1. Enable feed for 5,000+ XP users
2. Invite 10-20 beta testers to seed content
3. Monitor engagement, quality, sentiment
4. Iterate on algorithm, UX
5. Build moderation tools

**Success Metrics:**
- 50% of 5K+ users post within 7 days
- Avg 3+ posts per active user per week
- <5% reported posts (content quality)

---

### 6.2 Phase 2: Premium Creator Pilot

**Timeline:** Month 3-4
**Goal:** Onboard 3-5 real wrestlers/coaches

**Target Creators:**
- 1-2 retired pro wrestlers (e.g., Mick Foley, Edge)
- 1-2 indie wrestlers with online following
- 1-2 mental health professionals/coaches

**Launch Plan:**
1. Reach out with personalized pitches
2. Offer rev-share deals
3. Co-create first challenge with each
4. Cross-promote on their socials
5. Measure user acquisition

**Success Metrics:**
- 1,000+ new signups from creator promo
- 50%+ of users engage with creator content
- 1+ successful paid challenge

---

### 6.3 Phase 3: Full Launch

**Timeline:** Month 5-6
**Goal:** Scale to 10+ creators, 10K+ users

1. Open creator applications
2. Build creator marketplace
3. Launch monetization features
4. PR push (TechCrunch, Product Hunt)
5. Paid user acquisition campaigns

## Part 7: Open Questions & Decisions Needed

### Content & Community

1. **Daily Post Limits:**
   - Rookies: 0 posts
   - Veterans (5K+): 3 posts/day
   - Main Eventers (25K+): Unlimited
   - **OR** should veterans have unlimited too?

2. **Comment Permissions:**
   - Should rookies be able to comment?
   - **Proposed:** 1,000 XP minimum to prevent spam
   - **OR** let anyone comment but rate-limit rookies?

3. **Moderation:**
   - Human moderators or AI-first?
   - Community voting on flagged content?
   - Should premium creators have mod powers?

4. **Content Ownership:**
   - Who owns posted promos?
   - Can KAYFABE use them in marketing?
   - Clear TOS needed

### Creator Economy

5. **Creator Verification:**
   - What proof do we need for wrestlers?
   - Should we verify coaches/therapists?
   - Background checks needed?

6. **Revenue Share:**
   - 70/30 split fair for creators?
   - Should split vary by tier (80/20 for partners)?

7. **Pricing:**
   - What's fair for paid challenges? ($9.99? $19.99?)
   - Monthly subscriptions to creators ($4.99 or $9.99)?
   - Platform subscription ($9.99 for Premium)?

### Technical

8. **Feed Algorithm:**
   - Chronological only (simple, fair)?
   - Engagement-based ranking (more addictive)?
   - ML recommendations (complex, personalized)?

9. **Notifications:**
   - Email for every pop? (too spammy?)
   - Push notifications for comments/follows?
   - Daily digest email instead?

10. **Scalability:**
    - How to handle viral posts (millions of pops)?
    - CDN for images/avatars?
    - Rate limiting strategy?
